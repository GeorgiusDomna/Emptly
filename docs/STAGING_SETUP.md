# Подготовка staging-среды

## Минимальные требования

- Отдельный домен/поддомен для frontend.
- Отдельный домен/поддомен для backend WS/HTTP.
- HTTPS и WSS обязательно.
- Отдельные env-переменные для staging.

## Что должно быть настроено

- CORS разрешает только staging frontend origin.
- Логирование включено на уровне `info`.
- Health endpoint доступен извне (`/health`).
- Ошибки отправляются в систему мониторинга.

## Проверка готовности

- Деплой выполняется из CI.
- Smoke-checklist (`docs/STAGING_SMOKE_CHECKLIST.md`) проходит полностью.
- Есть ответственный за подтверждение релиза.
