# ADR-0001: Базовая архитектура приватного чата

## Статус

Принято.

## Контекст

Нужно реализовать чат по ссылке для двух участников без хранения сообщений.
Проект должен быть простым для MVP и устойчивым к дальнейшему развитию.

## Решения

1. **Транспорт real-time:** WebSocket (`ws`).
2. **Серверный фреймворк:** Express.
3. **Источник истины по комнате:** только сервер.
4. **Хранение данных комнаты:** in-memory `Map<roomId, RoomState>`.
5. **Лимит участников:** максимум 2 клиента на комнату.
6. **Персистентность сообщений:** отсутствует.
7. **Контракт событий:** фиксированный, версия `v1` (см. `docs/WS_PROTOCOL.md`).
8. **Валидация входа:** все payload валидируются до бизнес-логики.
9. **Очистка состояния:** удаление клиента на disconnect + удаление пустой комнаты.
10. **Безопасность:** env-only секреты, CORS whitelist, лимиты размера сообщений.

## Последствия

### Плюсы

- Простая и быстрая реализация MVP.
- Минимальные издержки на инфраструктуру.
- Предсказуемое поведение комнат.

### Минусы

- Ограниченное горизонтальное масштабирование без дополнительного адаптера (например, Redis pub/sub).
- Состояние теряется при перезапуске процесса (для MVP это ожидаемо).

## Что делать при масштабировании

- Добавить shared adapter для маршрутизации между инстансами.
- Ввести sticky sessions или единый real-time слой.
