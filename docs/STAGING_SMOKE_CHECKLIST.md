# Staging smoke-checklist

## Подготовка

- [ ] Развернуты актуальные версии frontend и backend
- [ ] Проверены env-переменные
- [ ] Сервис доступен по HTTPS/WSS

## Базовый функционал комнаты

- [ ] Пользователь A открывает стартовую страницу
- [ ] Пользователь A создает комнату и копирует ссылку
- [ ] Пользователь B переходит по ссылке и подключается
- [ ] В комнате одновременно 2 участника

## Обмен сообщениями

- [ ] Сообщение A видно у B
- [ ] Сообщение B видно у A
- [ ] Время доставки визуально без заметных задержек

## Ограничения и ошибки

- [ ] Пользователь C не может войти (получает `room_full`)
- [ ] При разрыве соединения есть понятное сообщение в UI
- [ ] При выходе одного участника второй получает `peer_left`

## Изоляция и очистка

- [ ] Сообщения не видны в других комнатах
- [ ] После выхода обоих участников комната очищается на сервере
- [ ] После перезапуска сервера нет "старых" сообщений

## Финализация

- [ ] Нет критичных ошибок в логах и Sentry
- [ ] Smoke чеклист подписан ответственным
