# План чистки legacy-кода

## Цель

Оставить в проекте только код, связанный с приватным чатом по ссылке.

## Что проверить и удалить/архивировать

- Старый CRUD/Mongo API, если не используется в текущем продукте.
- Экспериментальные файлы и демо-скрипты без применения в MVP.
- Временные бинарные файлы/изображения, не относящиеся к UI проекта.
- Большие текстовые заглушки и тестовые дампы.

## Безопасная процедура

1. Составить список кандидатов на удаление.
2. Подтвердить, что на них нет импортов из runtime-кода.
3. Удалять по частям в отдельных PR.
4. После каждого удаления прогонять smoke-тест.

## Критерий завершения

- В репозитории нет неиспользуемых экспериментальных артефактов.
- В `server` и `client` остался только код, поддерживающий текущий продуктовый поток.
