# Командный workflow

## Ветки

- `main` (или `master`) — только стабильный код
- feature-ветки: `feat/<краткое-имя>`
- bugfix-ветки: `fix/<краткое-имя>`

## Pull Request

- Один PR = одна логическая задача.
- Мелкие, обозримые PR предпочтительнее крупных.
- Перед ревью автор проверяет локально:
  - сборку
  - тесты
  - ключевой сценарий чата

## Ревью

- Минимум 1 аппрув до merge.
- Для изменений протокола обязателен просмотр фронтенда и бэкенда.
- Блокирующие замечания закрываются до merge.

## Merge политика

- Merge только при зеленом CI.
- Никакого bypass обязательных проверок.
